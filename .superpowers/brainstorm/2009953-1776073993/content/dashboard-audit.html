TinkerClaw Dashboard Audit

14/15 API endpoints working. 6 bugs found. What should we fix first?

Critical Bugs (must fix)

1

Notes Tab Completely Broken (404)

/api/notes returns 404 — the Notes tab shows nothing. Voice server has notes at /api/notes but the dashboard proxy route doesn't reach it. Entire Notes feature is dead.

2

Config POST Goes Nowhere

The "Apply" button in Overview pipeline config POSTs to /api/voice-config on the dashboard, not through the proxy to the voice server. Config changes don't actually apply.

High Priority Bugs

3

Missing CORS on Proxy Responses

Non-SSE proxy responses have no CORS headers. Cross-origin access fails for all non-streaming endpoints.

4

Tool Execute Double-Click

Execute button not disabled during API call. Users can fire duplicate requests by clicking twice.

UX Improvements

5

Add Loading States Everywhere

No spinners, no disabled buttons during fetches. Users have no idea if their click registered or if the system is working.

6

Show Errors to User

Config load failures, event polling errors — all silently swallowed. User sees stale data with no indication anything's wrong.

7

Add OTA Firmware Update Tab

Dragon has /api/ota/check and /api/ota/firmware.bin endpoints but no dashboard UI to trigger updates.

8

Device Config Push

No way to change device WiFi, voice mode, or LLM model from the dashboard. Currently requires physical Tab5 interaction.
